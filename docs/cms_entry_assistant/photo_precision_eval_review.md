**Q1. トピック適合のギャップ**  
現状観察: 多い順に、国・地域ドリフト（中国記事に日本人/日本医師、戦争記事に無関係な他国旗）、汎用抽象語ドリフト（経済→株価、建設→都市景観、結婚→現代指輪）、前slot/親記事語の使い回し（健康全slotが背中、果物記事でみかん固定）。  
修正提案: `article_topic / country / key_entities / slot_heading` を必須アンカー化し、文脈外の国名・都市名・人物属性は強デモート。

**Q2. 抽象 / 具体のバランス**  
現状観察: Aランドマークは比較的良いが時代ズレあり。B国旗は他国旗混入で危険。C抽象はグラフ/シルエット/3D過多。D行動は構図は良いが国籍・年齢がズレる。E産業は無難だが汎用都市に流れる。  
修正提案: 改善優先は `B > C > E/D`。Bは国ペア完全一致、Cは抽象でも固有トピック必須、E/Dは地域・年齢・職業のmust条件を足す。

**Q3. `primary_query` の質**  
現状観察: `シニア 背中 姿勢` や `みかん 果物 食卓` の重複で、slot固有の論点が検索に反映されていない。これはrerank以前の母集団品質の問題。  
修正提案: slotごとに別クエリ生成へ仕様変更すべき。`hero=タイトル核`、`h4=見出し固有語+記事トピック+視覚名詞` とし、同一queryは禁止、重複時はh4語で再生成。

**Q4. hard-block 拡張候補**  
現状観察: 例は、中国h4_1の`3dレンダリング/白背景に隔離`、教育heroの`小学生セックス`、教育h4_1の`赤ちゃん`、中国h4_2の`日本人医師/X線`、戦争h4_2の`京都/紅葉/家族/カップル`。  
修正提案: hard-blockは `セックス`, 文脈外の`赤ちゃん/乳児`, 文脈外国名・都市名・国旗, 非医療slotの`X線/患者/医師`。soft-demoteは `3dレンダリング`, `白背景`, `隔離`, `観光`, `紅葉`, `家族`, `カップル`, `金融グラフ`。

**Q5. 優先順位**  
現状観察: 最重要はQ3。hard-blockを増やすより、slot別queryで母集団を直す方がQ1/Q2/Q4を同時に改善できる。  
修正提案: 1 commitなら `docx_parser` / `istock_search` の `build_suggestion` 周辺で実装。手順は、1. slot入力に`topic/country/entities/h4_title`を渡す、2. プロンプトでslot別`primary_query`と重複禁止を明記、3. 生成後に同一query・必須アンカー欠落を検出、4. h4固有語で再生成/補正、5. 健康重複・中国日本人混入・戦争他国旗の回帰テストを追加。
